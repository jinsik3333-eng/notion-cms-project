# ResumeLens - 자소서 AI 분석 서비스 MVP PRD

## 🎯 핵심 정보

**목적**: 구직자가 작성한 자소서를 Claude AI로 5가지 관점(논리구조, 직무적합성, 차별성, 문장력, 면접관 시선)에서 동시 분석하여 개선점을 제시

**사용자**: 입사 면접을 준비하는 20~40대 직장인 및 신입 구직자

---

## 🚶 사용자 여정

```
1. 랜딩 페이지 (서비스 소개)
   ↓ [자소서 분석 시작 버튼 클릭]

2. 자소서 분석 페이지
   ↓ [자소서 텍스트 입력]
   ↓ [분석 시작 버튼 클릭]

3. 분석 진행 중 (로딩)
   ↓ [5가지 관점 동시 분석 중...]

4. 분석 결과 페이지
   ↓ [5가지 분석 결과 확인]

   [추가 탐색 경로]
   ├─ 서비스 소개 페이지 (Notion CMS) → 다시 분석하기
   ├─ 가격표 페이지 (Notion CMS)
   └─ 후기 페이지 (Notion CMS)

5. 완료
   ↓ [새로운 자소서 분석 또는 페이지 이동]
```

### 분석 결과 페이지

> **구현 기능:** `F002`, `F003`, `F011` | **메뉴 위치:** 자소서 분석 후 자동 이동

| 항목 | 내용 |
|------|------|
| **역할** | 5가지 관점의 AI 분석 결과를 구조화된 형식으로 표시 |
| **진입 경로** | 자소서 분석 페이지에서 분석 완료 후 자동 이동 |
| **사용자 행동** | 5가지 분석 결과(점수, 피드백, 개선 제안)를 확인하고 다음 액션 선택 |
| **UI 구조** | **1. 종합 평가 섹션** (상단)<br>• 원형 진행바: 종합 점수 (0-100) 시각화<br>• 점수 옆 평가 문구: "우수", "양호", "보통", "개선필요" (점수 대역별)<br>• 종합 요약 텍스트 (200-300자)<br><br>**2. 5가지 분석 결과 카드** (그리드 또는 탭)<br>각 카드에 포함:\n  - 아이콘 + 제목 (논리구조, 직무적합성, 차별성, 문장력, 면접관 시선)<br>  - 점수 (0-100, 컬러 코드: 빨강<60, 노랑 60-79, 초록 80+)<br>  - 피드백 텍스트 (200-300자)<br>  - 개선 제안 목록 (3개 항목, 각 50-100자)<br><br>**3. 에러 처리**<br>• 분석 실패 시: 에러 메시지 + "다시 시도" 버튼<br>• 타임아웃 (60초 이상): "시간이 오래 걸리고 있습니다" + 취소 버튼 |
| **주요 기능** | • 분석 완료 메시지<br>• 원형 차트로 종합 점수 시각화<br>• 5가지 관점별 분석 결과 카드 (색상 코딩: 0-59 빨강, 60-79 노랑, 80+ 초록)<br>• 각 항목별 "자세히 보기" 확장 기능 (선택사항)<br>• 분석 완료 시간 표시<br>• **다시 분석하기** 버튼 (새 자소서)<br>• **홈으로 돌아가기** 버튼<br>• 결과 공유 기능 (링크 복사, 선택사항 - MVP 이후) |
| **다음 이동** | 다시 분석하기 → 자소서 분석 페이지, 홈으로 → 랜딩 페이지 |

```typescript
{
  id: string;           // 분석 세션 ID (UUID)
  resumeText: string;   // 원본 자소서 (500-5000자)
  analyses: {
    logicStructure: {
      score: 0-100;       // 논리구조 점수
      feedback: string;   // 피드백 (200-300자)
      suggestions: string[]; // 개선 제안 (2-3개)
    };
    jobSuitability: {
      score: 0-100;
      feedback: string;
      suggestions: string[];
    };
    differentiation: {
      score: 0-100;
      feedback: string;
      suggestions: string[];
    };
    writingQuality: {
      score: 0-100;
      feedback: string;
      suggestions: string[];
    };
    interviewerPerspective: {
      score: 0-100;
      feedback: string;
      suggestions: string[];
    };
  };
  overallScore: 0-100;  // 5가지 점수의 평균
  summary: string;      // 종합 평가 (200-300자)
  analyzedAt: ISO8601;  // 분석 완료 시간
}
```

## ⚙️ API 통합 & 에러 처리 전략

### Claude API 통합

**엔드포인트**: `/api/analyze-resume`

**요청 본문**:
```json
{
  "resumeText": "자소서 텍스트 (50-5000자)",
  "analysisType": "comprehensive" // 5가지 관점 동시 분석
}
```

**응답 형식**: 위 "자소서 분석 결과 스키마" 참조

**에러 처리**:
- **429 (Rate Limited)**: 30초 후 자동 재시도
- **500+ (API 오류)**: 사용자 친화적 메시지 표시 + "다시 시도" 버튼
- **타임아웃 (60초)**: "분석이 오래 걸리고 있습니다" 메시지

### Notion API 통합

**데이터 조회 주기**: 애플리케이션 시작 시 + 페이지 방문 시 캐싱

**캐싱 전략**:
- **PricingPlans**: 24시간 캐시 (변경 빈도 낮음)
- **Testimonials**: 12시간 캐시 (변경 빈도 낮음)
- **ContentPages**: 6시간 캐시 (변경 빈도 중간)
- 캐시 무효화: 서버 재시작 시 자동 초기화

**에러 처리**:
- **403 (인증 실패)**: 환경 변수 확인 + 에러 로그 기록
- **404 (DB 없음)**: 기본 데이터 반환 + 관리자 알림
- **네트워크 오류**: 마지막 캐시된 데이터 사용 (Stale-While-Revalidate)

### 보안 & 리소스 관리

**Claude API 비용 관리**:
- 사용자당 1일 최대 5회 분석 (선택사항, MVP 이후 적용)
- 분석 결과는 서버에 저장하지 않음 (사용자 브라우저에만 표시)

**Notion API 인증**:
- API 키는 환경 변수에 저장 (.env.local)
- 서버사이드에서만 호출 (클라이언트 노출 방지)
